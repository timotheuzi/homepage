Dumb Phone User Guide

Welcome to Dumb Phone! This guide will help you understand and use the features of this network security application to protect your device and privacy.

1. Introduction

Dumb Phone is designed to give you complete control over your device's network traffic. It uses Android's VPN Service API to intercept, analyze, and filter all outgoing and incoming packets.

Important: When you first activate a firewall mode, Android will ask for VPN Permission. You must grant this for the application to function.

2. Dashboard: Security at a Glance

The Dashboard is your main control center.

  • Firewall Status: Shows which modes are currently active (Dumb Phone Mode, Soft Firewall Mode, or Both).
  • Quick Insights: Real-time counters for blocked connections and allowed connections.
  • Recent Activity: A quick preview of the last few network connections handled by the firewall.

Toggling Firewall Modes

Each firewall mode has an Activate/Deactivate button:

  • Tap Activate Now to start the firewall mode
  • Tap Deactivate to stop the firewall mode
  • The button will be disabled while the toggle operation is in progress to prevent multiple simultaneous toggles

3. Firewall Modes

Dumb Phone Mode

This is the most restrictive mode. It follows a "Block All by Default" policy.

  • What it does: Blocks all network traffic.
  • Exceptions: It automatically allows traffic essential for Voice over IP (VoIP) and text messaging (SIP, SMTP/IMAP ports).
  • When to use: Use this when you want maximum privacy and want to ensure no background apps are communicating with the internet.

Soft Firewall

A more flexible protection mode.

  • What it does: Allows traffic by default but filters it against your custom rules and the country blacklist.
  • When to use: Use this for everyday protection without breaking most applications.
  • How it works:
    1. First checks if the connection matches any custom firewall rules
    2. Then checks if the destination IP is in any active country blacklist
    3. If no rules match, the connection is allowed by default

4. Logs and Activity

The Logs screen provides a detailed history of every connection.

  • Blocked entries appear in red.
  • Allowed entries appear in green.
  • Each log shows the destination IP, port, protocol, and the time of the event.
  • You can search logs by IP address or filter them by date.
  • Tap on any log entry to see detailed information including source IP, process name, and package name.

Log Filtering

  • IP Filter: Enter an IP address to show only logs matching that IP
  • Date Range: Select a start and end date to filter logs by time period
  • Clear Filters: Remove all active filters to see all logs

5. Blocklists and Countries

You can block entire countries to prevent any communication with servers located there.

  • Navigate to the Blocklist section.
  • Toggle countries on/off to instantly update the firewall rules.
  • The app uses a pre-loaded database of IP ranges for each country.
  • Changes take effect immediately without waiting for the next refresh cycle.

Adding Countries

  1. Tap the Add Country button
  2. Enter the country code (e.g., CN, RU, US)
  3. Enter the country name
  4. Select a threat level (Low, Medium, High, Critical, Malicious, Suspicious)
  5. Optionally add custom IP ranges in CIDR format
  6. Tap Add to List

Viewing Country Details

Tap on any country in the blocklist to see:

  • Threat level
  • Status (Active/Inactive)
  • Number of blocked connections
  • Number of loaded IP ranges
  • List of blocked IP ranges

Removing Countries

  1. Tap on a country to open details
  2. Tap Remove to delete the country from the blocklist
  3. The firewall rules will be refreshed immediately

6. Firewall Rules

Create custom rules to allow or block specific traffic.

Creating a Rule

  1. Navigate to the Rules section
  2. Tap Add Rule
  3. Select the rule type:
    • IP Address: Block/allow traffic to/from a specific IP
    • Port: Block/allow traffic on a specific port
    • Protocol: Block/allow traffic using a specific protocol (TCP/UDP)
  4. Enter the value (IP address, port number, or protocol)
  5. Optionally specify a port and protocol
  6. Select the action: Allow or Block
  7. Tap Save Rule

Managing Rules

  • Toggle Rules: Use the switch to enable/disable rules without deleting them
  • Delete Rules: Tap the delete icon to permanently remove a rule
  • View Rule Details: See all rule parameters at a glance

7. Settings

  • Log Retention: Choose how many days of logs to keep (1-30 days). Older logs are automatically purged to save space.
  • Clear Logs: Manually delete all recorded activity.
  • Update Database: Refresh the threat intelligence and country IP range data.

Privacy and Security

Privacy Note: All processing and logging happen locally on your device. Your network traffic data is never sent to our servers.

Security Features:

  • VPN-based traffic interception
  • Real-time packet filtering
  • Country-based IP blocking
  • Custom rule engine
  • Local-only data storage

Troubleshooting

VPN Permission Issues

If the app cannot start the firewall:

  1. Ensure you granted VPN permission when prompted
  2. Check that no other VPN app is running
  3. Try restarting the app

Rules Not Taking Effect

If your rules don't seem to work:

  1. Ensure the rule is enabled (toggle is on)
  2. Check that the rule type and value are correct
  3. For country blacklists, ensure the country is marked as Active
  4. The firewall refreshes rules every 30 seconds, or immediately when you make changes

High Battery Usage

The VPN service runs continuously when active. To reduce battery usage:

  1. Use Soft Firewall mode instead of Dumb Phone mode when possible
  2. Deactivate the firewall when not needed
  3. Reduce the number of active rules
